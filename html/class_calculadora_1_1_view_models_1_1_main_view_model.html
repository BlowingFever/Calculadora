Detailed Description

Root ViewModel that owns the active view and exposes navigation commands. WPF resolves the correct DataTemplate for CurrentView automatically via type-keyed templates registered in App.xaml.

Child ViewModels are created as singletons so that their state (calculation history, graph expressions, etc.) is preserved when the user switches between views and returns.
