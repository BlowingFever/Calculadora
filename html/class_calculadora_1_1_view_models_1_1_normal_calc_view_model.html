Detailed Description

ViewModel de la calculadora estándar (no gráfica). Implementa un evaluador de expresiones infijas con precedencia de operadores mediante dos listas de tokens paralelas: una para el cómputo y otra para el historial visual que se muestra encima del display.

Algoritmo de evaluación:

  1. Cada dígito se acumula en Display.
  2. Al pulsar un operador, el display se añade a _tokens y se llama a CollapseTokens para reducir cualquier operación pendiente de mayor o igual precedencia.
  3. Al pulsar =, se fuerza la reducción completa (precedencia 0) y se muestra el resultado.

La pila _expressionTokens nunca se colapsa: sólo sirve para construir la cadena de historial que ve el usuario.
